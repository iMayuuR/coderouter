Features

Why CodeRouter?

Same Claude Code workflow — any model you choose behind it. One OpenRouter key, a tiny local proxy (text + smart vision routing), global skills and MCP, secrets in .env. Swap GPT, Gemini, Llama, or free tiers without changing how you work in the terminal.

🌐

200+ models

GPT-4.1, Gemini 3.1 Pro, Llama 4, DeepSeek, and the rest of OpenRouter’s catalog — one endpoint.

👁️

Smart vision proxy

Detects images in prompts and routes to a vision-capable model (e.g. Qwen VL) so the terminal agent does not break on multimodal input.

🧠

200+ agentic skills

Brainstorming, TDD, debugging, code review, UI/UX, browser automation, security — installed globally with the setup scripts.

🔌

MCP integrations

Optional @21st-dev/magic: with MAGIC_API_KEY in .env, the installer adds user-scope MCP under ~/.claude.json → top-level mcpServers (all folders). Run Claude Code once first so that file exists; restart the app after install.

🌍

Global claude / coderouter

After install-global-skills.js (or setup-agent.ps1 with ai-agent-config.json), run claude or coderouter from any folder — after a full terminal restart.

🆓

Free tiers

Many strong free models on OpenRouter; pay only for what you route. Keys live in .env (gitignored).

Quick start

~60 seconds to Claude Code

Matches QUICKSTART.md — five beats from clone to a global claude command.

Before you start: Install Node.js (LTS, includes npm). Install Claude Code (npm install -g @anthropic-ai/claude-code or the official installer). You need an OpenRouter API key.

📥 1

Clone & install

git clone https://github.com/iMayuuR/coderouter.git cd coderouter npm install
📄 2

Create .env

Windows copy .env.example .env copy install-global-skills.example.env .env
macOS / Linux cp .env.example .env cp install-global-skills.example.env .env

Set OPENROUTER_API_KEY and CLAUDE_MODEL (e.g. openrouter/free). Optional: MAGIC_API_KEY for @21st-dev/magic — see QUICKSTART.md. Never commit .env.

⚡ 3

One-shot setup

Recommended (any OS) node install-global-skills.js

Skills, PATH, aliases, and Windows user env always apply. Optional MCP (MAGIC_API_KEY): user-scope ~/.claude.json → mcpServers — open Claude Code once if that file is missing, then re-run this script. Restart Claude after MCP changes.

Windows only (optional) .\setup-agent.ps1

Needs ai-agent-config.json with user_preferences; runs npm install and the same global installer.

🔄 4

Restart terminal

Close every terminal window, then open a new one so your shell profile and PATH reload.

🚀 5

Run anywhere

claude or coderouter

CodeRouter launcher (proxy + OpenRouter). In this repo: npm run claude — do not use plain npx claude; it skips the proxy.

!

Restart required. If claude / coderouter is not found, you skipped a full terminal restart — close every window, open a fresh one, try again.

Let an AI agent run the setup

Copy the prompt below and paste it into Cursor, Claude, Windsurf, or any coding agent. It mirrors QUICKSTART: clone (or use an existing repo), npm install, create .env from .env.example (or install-global-skills.example.env) with your OpenRouter key and model, run node install-global-skills.js (or .\setup-agent.ps1 on Windows only if you use ai-agent-config.json), then insist on a full terminal restart before using claude or coderouter.

Copied to clipboard.

Optional: ai-agent-config.json

Agents can read this file for your key and model. Edit user_preferences, then share the file plus the copied setup prompt.

  1. Credentials Set api_key and model (example: openrouter/free).
  2. Install & env Agent runs npm install and creates .env from .env.example.
  3. How you launch Per-session: Windows .\run-claude.ps1 · Unix ./run-claude.sh. After global setup + restart: use claude or coderouter anywhere.
ai-agent-config.json 
{
  "user_preferences": {
    "api_key": "sk-or-v1-...",
    "model": "openrouter/free",
    "is_default": true
  },
  "agent_actions": {
    "install_dependencies": "npm install",
    "configure_env": ".env from .env.example + user_preferences",
    "launch_on_windows": ".\\run-claude.ps1",
    "launch_on_unix": "./run-claude.sh"
  }
}

Base URL override

Compatible API

OpenRouter picks healthy providers; you switch models by editing .env, not by reinstalling the agent.
